<argument> ARGUMENT

Words starting with "<", ending with ">" and upper-case words are interpreted as positional arguments.

<argument> ARGUMENT

    
    Usage: mi_programa <host> <port>

-o --opcion

Words starting with "<", ending with ">" and upper-case words are Words starting with one or two dashes (with exception of "-", "--" by themselves) are interpreted as short (one-letter) or long options, respectively.

-o --opcion

Short options can be "stacked" meaning that -abc is equivalent to -a -b -c.

-o --opcion

Long options can have arguments specified after space or equal "=" sign: --input=ARG is equivalent to --input ARG. Short options can have arguments specified after optional space: -f FILE is equivalent to -fFILE.

command

Todas las palabras que no sigan la convencion de argumentos u opción se interpretan como comandos o subcomandos

(Elementos requeridos)

Todos los elementos son requeridos por default al menos que sean incluidos entre "[ ]" , por ejemplo cuando se necesita un grupo de elmentos mutualmente exclusivos.

        
        Usage: my_program (--either-this <and-that> | <or-this>)
        
        




    
(Elementos requeridos)


        
Otro caso de uso es cuando se necesita especificar que si un elemento
        está presente entonces el siguiente también debe ser requerido


        
        
        Usage: my_program [( <un-argumento> <otro-argumento>)]
        
        




    
elemento | otro


        
Los elementos mutualmente exclusivos pueden ser separados con un pipe "|"


        
        
        Usage: my_program go (--up | --down | --left | --right)
        
        





    
elemento | otro



Usar paréntesis "( )" para agrupar elementos cuando uno de los casos mutualmente exclusivos es requeridos. Usar corchetes "[ ]" para agrupar elmento cuando ninguno de los elementos mutualmente exclusivos es requerido.


        
        
		Usage: my_program go [--up | --down | --left | --right]
        
        





    
elemento...



Usar tres puntos suspensivos "..." para especificar que un argumento o grupo de elementos de la izquierda puede ser repetido una o mas veces.


        
        
		Usage: my_program abrir <file>...
       	       my_program mover (<from> <to>)...
        
        




    
[options]



[options] es un atajo que permite poder evitar listar todas las opciones (de una lista de opciones con desripciones) en un patrón









[options]


Esto

        
        
		Usage: my_program [options] 

		--all             List everything.
		--long            Long output.
		--human-readable  Display in human-readable format..
        
        





[options]


Es equivalente a esto

        
        
Usage: my_program [--all --long --human-readable] 

--all             List everything.
--long            Long output.
--human-readable  Display in human-readable format.
        
        






[- -]


Dos guiones medios,"- -",cuando no son parte de una opción, suelen ser usados como una convención para separar opciones y argumentos posicionales, por ejemplo cuando nombres de archivos pueden ser confundidos con opciones.

 
        
        
		Usage: my_program [options] [--] ...
        
        





[-]


Un solo "-", (fuera de las opciones) es usado por convención para decirle a un programa que debe leer por stdin en lugar de un archivo. Para esta convención deberás agregar "[-]". "-" por si sólo puede usarse como un comando normal con cualquier significado

 
        
        
		Usage: my_program read ([-] | file )
        
        






Descripción de opciones


La descripción de opciones es una lista de opciones que se ponen debajo de los patrones de uso para alguno de los siguientes casos.



    

  • Sinónimos de opciónes cortas y largas.

  • Que una opción tiene un argumento.

  • Valor por defecto de un argumento de una opción.








Descripción de opciones


Cada línea que comienza con "-" o "--" sin contar espacios es tratada com una descripción de opciones.


        
        
		Usage: my_program read ([-] | file )
        
        








Descripción de opciones


To specify that an option has an argument, put a word describing that argument after a space (or equals "=" sign) as shown below. Follow either  or UPPER-CASE convention for options' arguments. You can use a comma if you want to separate options. In the example below, both lines are valid, however it is recommended to stick to a single style.



        
        
		-o FILE --output=FILE       # without comma, with "=" sign
		-i <file>, --input <file>   # with comma, without "=" sign
        
        






Descripción de opciones

        
        
		-o FILE --output=FILE       # without comma, with "=" sign
		-i <file>, --input <file>   # with comma, without "=" sign
        
        






Descripción de opciones


Usa dos espacios para separar opciones con su descripcion informal

        
        
--verbose MORE text.    # BAD, will be treated as if verbose
                        # option had an argument MORE, so use
                        # 2 spaces instead
-q        Quit.         # GOOD
-o FILE   Output file.  # GOOD
--stdout  Use stdout.   # GOOD, 2 spaces
        
        






Descripción de opciones


Para setear un valor por defecto para una opcion con un argumento. poner el argumento en la descipción de la opción en la forma [default: <the-default-value>].


        
        
--coefficient=K  The K coefficient [default: 2.95]
--output=FILE    Output file [default: test.txt]
--directory=DIR  Some directory [default: ./]

API


Muy sencillo de usar


        
        
		docopt(doc, argv=None, help=True, 
			version=None, options_first=False)
        
        






API


    

  •  argv es un vector argumento opcional. por defecto toma sys.argv[1:]. Alternativamente se puede pasar una lista de strings como ['--verbose', '-o', 'hai.txt'].


  •  help default True. Especifica si el parser debe automaticamente y terminar en caso de -h o --help .


  •  version por defecto None, es un argumento opcional que especifica que si cuando se pasa en la linea de comando debe imprimir la versión y cerrarse.


  •  options_first por defecto False, si se especifica True no dejará mezclar opciones y argumentos posicionales. Luego del primer argumento posicional los demas serán interpretados como posicionales aunque luzcan como opciones.


  •  return value es un diccionario con las opciones, argumentos y comandos como claves, exactamente como fueron descriptas en el help.

 









"""Naval Fate.

Usage:
  naval_fate.py ship new <name>...
  naval_fate.py ship <name> move <x> <y> [--speed=<kn>]
  naval_fate.py ship shoot <x> <y>
  naval_fate.py mine (set|remove) <x< >y> [--moored | --drifting]
  naval_fate.py (-h | --help)
  naval_fate.py --version

Options:
  -h --help     Show this screen.
  --version     Show version.
  --speed=<kn>  Speed in knots [default: 10].
  --moored      Moored (anchored) mine.
  --drifting    Drifting mine.

"""
from docopt import docopt

if __name__ == '__main__':
    arguments = docopt(__doc__, version='Naval Fate 2.0')
    print(arguments)

Docopt

  • Definir la interfaz de tu aplicación como linea de comando, help.
    • Basado en convenciones ampliamente usadas durante décadas
    • Autodocumentación, genera el HELP.

Veamos un ejemplo de HELP

    
        Usage:
          mi_programa comando --opcion <argumento>
          mi_programa [<argumento-opcional>]
          mi_programa --otra-opcion=<con-argumento>
          mi_programa (--o-esta-opcion | <o-esta-otra>)
          mi_programa <argumentos-repetitivos> <argumento-repetititvo>...

<argument> ARGUMENT

Las palabras empezando con "<", terminando con ">" y palabras en mayúsculas son interpretadas como argumentos posicionales

<argument> ARGUMENT

    
    Usage: mi_programa <host> <port>

-o --opcion

Son intepretadas como opciones las palabras empezando por "-" o "- -" (con la excepción de "-" y "--" por si mismos son intepretados como opciones)

-o --opcion

Las opciones cortas pueden encadenarse 

			
        		Usage: mi_programa -abc
        		
        		
			
        		Usage: mi_programa -a -b -c

-o --opcion

Las opciones largas pueden tener argumentos, especificados después de un espacio o un igual "=":

--input=ARG es equivalente a --input ARG.

Las opciones cortas pueden tener argumentos especificados luego de un espacio opcional:

-f FILE es equivalente a -fFILE.

command

Todas las palabras que no sigan la convencion de argumentos u opción se interpretan como comandos o subcomandos

(Elementos requeridos)

Todos los elementos son requeridos por default al menos que sean incluidos entre "[ ]" , pero por ejemplo cuando se necesita un grupo de elmentos mutualmente exclusivos se usan ( ).

			
        Usage: mi_programa (--either-this <and-that> | <or-this>)
        
        




    
(Elementos requeridos)


        
Otro caso de uso es cuando se necesita especificar que si un elemento
        está presente entonces el siguiente también debe ser requerido


        
        
        Usage: mi_programa [(<un-argumento> <otro-argumento>)]
        
        




    
elemento | otro


        
Los elementos mutualmente exclusivos pueden ser separados con un pipe "|"


        
        
        Usage: mi_programa go (--up | --down | --left | --right)
        
        




    
elemento...



Usar tres puntos suspensivos "..." para especificar que un argumento o grupo de elementos de la izquierda puede ser repetido una o mas veces.


        
        
		Usage: mi_programa open <file>...
       	       mi_programa mover (<from> <to>)...
        
        




    
[options]



[options] Es un atajo que permite poder evitar listar todas las opciones (de una lista de opciones con descripciones) en un patrón









[options]


Esto

        
        
		Usage: mi_programa [options]

		--long            Salida larga.
		--human-readable  Mostrar en formato human-readble..
        
        





[options]

API


Muy sencillo de usar


        
        
		docopt(doc, argv=None, help=True, 
			version=None, options_first=False)
        
        






API


    

  •  argv es un vector argumento opcional. por defecto toma sys.argv[1:]. Alternativamente se puede pasar una lista de strings como ['--input', '-o', 'archivo.txt'].


  •  help default True. Especifica si el parser debe mostrar el help automaticamente y terminar en caso de -h o --help .


  •  version por defecto None, es un argumento opcional que especifica que si cuando se pasa en la linea de comando debe imprimir la versión y cerrarse.


  •  options_first por defecto False, si se especifica True no dejará mezclar opciones y argumentos posicionales. Luego del primer argumento posicional los demas serán interpretados como posicionales aunque luzcan como opciones.


  •  return value es un diccionario con las opciones, argumentos y comandos como claves, exactamente como fueron descriptas en el help.

 









"""Batalla naval.

Usage:
  batalla_naval.py ship new <name>...
  batalla_naval.py ship <name> move <x> <y> [--speed=<kn>]
  batalla_naval.py ship shoot <x> <y>
  batalla_naval.py mine (set|remove) <x< >y> [--moored | --drifting]
  batalla_naval.py (-h | --help)
  batalla_naval.py --version

Options:
  -h --help     Show this screen.
  --version     Show version.
  --speed=<kn>  Speed in knots [default: 10].
  --moored      Moored (anchored) mine.
  --drifting    Drifting mine.

"""
from docopt import docopt

if __name__ == '__main__':
    arguments = docopt(__doc__, version='Batalla naval 2.0')
    print(arguments)
